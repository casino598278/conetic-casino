# Deployment Setup

How to get Conetic Casino running on Render and wired up to the Telegram bot.

---

## 1. Prerequisites

- GitHub repo connected (this repo: `casino598278/conetic-casino`)
- [Render](https://render.com) account
- Telegram bot created via [@BotFather](https://t.me/BotFather) → you have a `BOT_TOKEN` and `BOT_USERNAME`
- A TON hot wallet mnemonic (generate with `pnpm exec tsx infra/scripts/gen-hot-wallet.ts`)
- (Optional) a [toncenter.com](https://toncenter.com) API key — the free tier works but is rate-limited

---

## 2. Create the Render service

Render auto-detects `render.yaml` at the repo root. First deploy:

1. **New → Blueprint** in the Render dashboard
2. Point it at this GitHub repo, branch `main`
3. Render will parse `render.yaml` and propose a single web service + a 1 GB persistent disk mounted at `services/api/data/` (that's where the SQLite DB lives)
4. Click **Apply**

The service will fail to boot the first time because secrets aren't set yet — that's expected. Fix it in the next step.

### What `render.yaml` already handles

- Build: `corepack enable && pnpm install --frozen-lockfile=false && pnpm build`
- Start: `pnpm start` (runs `node services/api/dist/index.js`)
- Health check: `/health`
- Auto-deploy: **on** — every push to `main` rebuilds + redeploys
- Persistent disk for the DB (survives redeploys)
- Public env vars (ports, TON network, rake bps, bet limits) are baked in

---

## 3. Set the secret env vars (Render dashboard)

Open the service → **Environment** tab → set each of these:

| Key | Value | Notes |
|-----|-------|-------|
| `BOT_TOKEN` | *from BotFather* | e.g. `7123456:AA...` |
| `BOT_USERNAME` | *from BotFather* | no `@` |
| `HOT_WALLET_MNEMONIC` | 24 words from `gen-hot-wallet.ts` | space-separated |
| `TON_API_KEY` | toncenter key (optional) | improves rate limits |
| `PUBLIC_WEB_URL` | `https://<service>.onrender.com` | Render gives you this after first deploy |
| `CORS_ORIGIN` | same as `PUBLIC_WEB_URL` | |
| `JWT_SECRET` | auto-generated by Render | leave alone |

`PUBLIC_WEB_URL` has a chicken-and-egg: you don't know the URL until the service is created. So create it first (it'll fail health check), then come back and fill these in, then click **Manual Deploy → Deploy latest commit**.

---

## 4. Fund the hot wallet (testnet)

The wallet only gets deployed on-chain after its first outgoing transaction, but you need it funded so withdrawals work.

1. Find the address: once the service is up, hit `GET /health` or check logs — it prints the hot wallet address on boot
2. Send testnet TON to it via [testnet.tonhub.com](https://testnet.tonhub.com) or a faucet (`@testgiver_ton_bot`)
3. At least **1 TON** is sensible for gas headroom

For mainnet, change `TON_NETWORK=mainnet` and `TON_ENDPOINT=https://toncenter.com/api/v2/jsonRPC` in `render.yaml` (or override via dashboard).

---

## 5. Wire up the Telegram bot

Once Render says the service is live and `/health` returns 200:

### 5a. Point BotFather at the Mini App

Message [@BotFather](https://t.me/BotFather):

```
/setmenubutton
→ pick your bot
→ send the URL: https://<service>.onrender.com
→ button text: Play
```

The bot also calls `setChatMenuButton` on startup, so this is belt-and-suspenders — either works.

### 5b. Verify the bot is polling

Check Render logs for:

```
[bot] @your_bot_username polling
```

If you see `[bot] BOT_TOKEN missing — skipping`, the env var didn't propagate — redeploy.

### 5c. Test from Telegram

Open your bot in Telegram → tap the menu button (bottom-left) or send `/play`. The Mini App should open and load the arena.

Commands registered with Telegram:

- `/play` — open the arena
- `/mine` — open mining race
- `/balance` — show TON balance
- `/deposit` — get deposit address + memo
- `/withdraw` — link to Wallet tab
- `/topup` — demo-only, +25 TON, 30m cooldown
- `/demo` — admin-only, toggle demo mode for users

---

## 6. Deploying updates

Any push to `main` redeploys automatically. To force a redeploy without code changes: **Manual Deploy → Deploy latest commit**.

Zero-downtime isn't guaranteed on the starter plan — expect ~30s unavailability during redeploys. The SQLite DB is on a persistent disk so user balances, rounds, and ledger entries survive.

---

## 7. Troubleshooting

**Mini App shows "No Telegram initData"**
→ You opened the Render URL directly in a browser. Open it via the bot's menu button instead.

**Health check fails after deploy**
→ Check logs for missing env vars. Most common: `HOT_WALLET_MNEMONIC` not set, or `BOT_TOKEN` invalid.

**Deposits don't credit**
→ Check the user used the correct memo (`/deposit` shows it). Credits are polled every ~10s; check logs for `[deposit]` lines.

**Withdrawals stuck**
→ Hot wallet out of TON for gas, or toncenter rate-limited. Fund the wallet; consider a paid `TON_API_KEY`.

**DB wiped after redeploy**
→ The persistent disk isn't attached. In the Render dashboard, check **Disks** tab — should show `casino-data` mounted at `/opt/render/project/src/services/api/data`.

**Bot commands (`/topup`, `/demo`) don't appear in Telegram menu**
→ Telegram caches the command list. Force-close and reopen the chat, or re-run `/start`.

---

## 8. Going to mainnet

When you're ready:

1. Generate a **new** mainnet hot wallet (do not reuse the testnet one)
2. In Render → Environment, set:
   - `TON_NETWORK=mainnet`
   - `TON_ENDPOINT=https://toncenter.com/api/v2/jsonRPC`
   - `HOT_WALLET_MNEMONIC=<new mainnet mnemonic>`
3. Fund the new wallet with real TON (start small)
4. Redeploy
5. Test with a tiny deposit + withdraw round-trip before announcing
